# Гайд по кроссплатформенному переносу конфига (Keyd -> Windows/macOS)

Этот репозиторий содержит настройки для `keyd` (Linux). Чтобы получить аналогичный функционал на Windows или macOS, следуйте этим инструкциям.

---

## 1. Универсальный способ: Kanata (Рекомендуется)

[Kanata](https://github.com/jtroo/kanata) — это инструмент на Rust, который работает почти так же, как `keyd`. Он позволяет использовать один и тот же файл конфигурации на разных системах.

### Как настроить вручную:
1. Скачайте бинарный файл для вашей ОС из [GitHub Releases](https://github.com/jtroo/kanata/releases).
2. Создайте файл `config.kbd` (возьмите содержимое из `portability/kanata.kbd`).
3. Запустите в терминале:
   ```bash
   kanata --config portability/kanata.kbd
   ```
4. **Важно:** Чтобы Kanata работала на Windows, её нужно запускать от имени администратора. На macOS требуются разрешения в "Секретность и безопасность" (Accessibility).

---

## 2. Windows: AutoHotkey (AHK)

AutoHotkey — самый мощный инструмент для Windows. Он позволяет переназначать клавиши на лету.

### Как настроить вручную:
1. Скачайте и установите [AutoHotkey v2](https://www.autohotkey.com/).
2. Скачайте файл `portability/windows.ahk` из этого репозитория.
3. Просто дважды кликните по файлу `.ahk`, чтобы запустить его.
4. Чтобы скрипт работал всегда: Нажмите `Win+R`, введите `shell:startup` и перенесите ярлык на этот скрипт в открывшуюся папку.

---

## 3. macOS: Karabiner-Elements (Нативный способ)

На macOS `keyd` не работает. Рекомендуемый стандарт для переназначения клавиш на низком уровне — это Karabiner-Elements.

Мы подготовили готовый скрипт конфигурации, который копирует основные фичи этой раскладки (Caps Lock -> Ctrl/Esc, Right Shift -> Backspace, и Программистский верхний ряд).

### Как настроить:
1. Установите [Karabiner-Elements](https://karabiner-elements.pqrs.org/).
2. Скопируйте файл конфигурации из этого репозитория в папку Karabiner:
   ```bash
   mkdir -p ~/.config/karabiner/assets/complex_modifications/
   cp portability/mac_karabiner.json ~/.config/karabiner/assets/complex_modifications/
   ```
3. Откройте приложение **Karabiner-Elements**.
4. Перейдите в раздел **Complex Modifications** -> нажмите **Add rule**.
5. Вы увидите правила из группы "Programmer Top Row & Modifiers (like Keyd)". Нажмите **Enable** для всех нужных правил.
6. *Примечание для русского языка:* Karabiner работает на уровне скан-кодов, поэтому для глубокой интеграции с русским языком на маке рекомендуется использовать стандартную раскладку "Русская - ПК" или создать свою через [Ukelele](https://software.sil.org/ukelele/).

## Сравнение инструментов

| Функция | Linux (keyd) | Windows (AHK) | macOS (Karabiner) | Universal (Kanata) |
| :--- | :--- | :--- | :--- | :--- |
| **Слои (Layers)** | Да | Да | Да | Да |
| **Tap-Hold (Overload)** | Да | Да (через скрипт) | Да | Да |
| **Сложные макросы** | Да | Очень мощно | Средне | Да |
| **Простота установки** | Высокая | Высокая | Высокая | Средняя |

## Совет по портативности

Если вы часто меняете системы, самый надежный способ — **Kanata**. Файл `kanata.kbd` максимально приближен к синтаксису `keyd`.

### Как синхронизировать изменения:
- При изменении `default.conf` для Linux, не забудьте вручную обновить `kanata.kbd`, чтобы ваши "мышечные привычки" не страдали при переходе на Windows/macOS.
- Не пытайтесь автоматизировать установку через скрипты (bash/powershell), так как системные пути и права доступа на разных ОС постоянно меняются. Ручное копирование конфига один раз в полгода — самый надежный метод.
